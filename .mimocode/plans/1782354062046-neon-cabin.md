# Plan: Multi-Card Flashcard Output Format

## Goal
Replace the single front/back card model with a multi-card system where the LLM generates 4-6 complementary learning cards per vocabulary word. Each card has a distinct type (translation, cloze, definition, example, synonym, pronunciation). FSRS scheduling stays per-word. Existing data is auto-migrated.

## Key Decisions
- **FSRS**: Per-word scheduling (all cards for a word share one schedule)
- **Migration**: Auto-migrate existing cards into a `LearningCard` list with `type="definition"`
- **Serialization**: `dataclass` + `json` (matches existing `dataclass`-heavy codebase)
- **Storage**: Add `learning_cards TEXT` column to `cards` table (JSON array)

## Step 1: New Data Model (`models.py`)

Add after existing enums (around line 28):

```python
class CardType(StrEnum):
    TRANSLATION = "translation"
    CLOZE = "cloze"
    EXAMPLE = "example"
    DEFINITION = "definition"
    SYNONYM = "synonym"
    PRONUNCIATION = "pronunciation"
    LISTENING = "listening"
    IMAGE = "image"
```

Add new dataclasses (after `WordSuggestion`):

```python
@dataclass(frozen=True)
class CardFace:
    prompt: str   # what the learner sees first (front of card)
    answer: str   # what is revealed (back of card)

@dataclass(frozen=True)
class LearningCard:
    card_type: str  # CardType value
    front: CardFace
    back: CardFace
```

Keep `NewCard` as-is for backward compat. Add a new field:

```python
@dataclass(frozen=True)
class NewCard:
    word: str
    definition: str
    example_sentence: str
    part_of_speech: str = ""
    source: str = "manual"
    learning_cards: tuple[LearningCard, ...] = ()
```

Using `tuple` instead of `list` because frozen dataclasses need immutable defaults.

Add helper functions:
- `learning_cards_to_json(cards: list[LearningCard]) -> str`
- `learning_cards_from_json(raw: str) -> list[LearningCard]`

## Step 3: Storage Migration (`storage.py`)

### Schema v2 → v3
```sql
ALTER TABLE cards ADD COLUMN learning_cards TEXT NOT NULL DEFAULT '[]';
```

### Auto-migration for existing rows
After the ALTER TABLE, run:
```sql
UPDATE cards SET learning_cards = json_array(
  json_object('card_type','definition',
    'front', json_object('prompt', word, 'answer', part_of_speech),
    'back', json_object('prompt', definition, 'answer', example_sentence)
  )
) WHERE learning_cards = '[]';
```

This wraps existing data into a single `definition` card:
- `front.prompt` = word (what learner sees)
- `front.answer` = part_of_speech (hint)
- `back.prompt` = definition (the answer)
- `back.answer` = example_sentence (extra context)

### Update `CardRepository.add()`
Serialize `learning_cards` from `NewCard.learning_cards` to JSON and store.

### Update `_card_from_row()`
Deserialize `learning_cards` from JSON column.

### Update `CardRecord`
Add `learning_cards: tuple[LearningCard, ...]` field.
